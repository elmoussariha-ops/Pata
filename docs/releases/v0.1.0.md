# Pata v0.1.0 — First public release

Release date: **2026-04-13**

## Why this release matters

`v0.1.0` establishes a public, reproducible baseline for a Rust-first agent platform focused on technical transparency:
- persona specialization,
- explicit reasoning and verification,
- traceable execution outputs,
- deterministic evaluation workflow.

## Highlights

### 1) Persona-specialized runtime
Pata ships with four personas (`developer`, `teacher`, `personal`, `smb`) wired through a shared orchestration core.

### 2) Visible verification flow
Every run follows an inspectable path:
`memory retrieval -> structured reasoning -> local/global verification -> final answer + trace`.

### 3) Reproducible public proof
You can reproduce the same launch evidence with local commands:

```bash
./quickstart/flagship-demo/run.sh --offline
python3 benchmarks/public-comparison/run_benchmark.py
cargo run -p cli -- --config config/app.toml --eval-suite evals/suite.v1.json --eval-output /tmp/pata-evals.json
```

## Scope and non-goals (honest state)

Included now:
- modular Rust workspace with clear crate boundaries,
- CLI and minimal server runtime,
- quickstart, benchmark, eval documentation,
- public launch artifacts and feedback loop templates.

Not included yet:
- enterprise-grade multi-tenant persistence,
- advanced DAG/ToT multi-branch orchestration,
- external judge model pipelines,
- full UI dashboard.

## Upgrade / adoption guidance

For new users:
1. Run the quickstart (`--offline`) to see a complete traceable run.
2. Generate benchmark report for shareable public proof.
3. Run eval suite to validate contracts and regression surface.

For contributors:
- start at `CONTRIBUTING.md`,
- use `examples/multi_persona_demo.md` and `USE_CASE_PROOFS.md` as product context,
- open issues with concrete reproduction steps.
